苹果将开放更多 iPhone X 面部数据给开发者；CakePHP 3.5.5 发布

2017-02-07 18:13:32 admin 2016

继最新的 Ubuntu 17.10 默认使用 GNOME Shell 桌面环境后，Canonical 昨天宣布已加入 GNOME 基金会的咨询委员会，双方更深入合作。GNOME 咨询委员会是什么？GNOME 将其解释为“通过提供资金和专家咨询来支持 GNOME 项目的利益相关者的组织”，其董事会成员还包括 Google、文档基金会、红帽和 SUSE 。

1、基于任务的并行处理框架 Jug 1.6.4 发布

Jug 1.6.4 发布了。Jug 是一个基于任务的并行处理框架，采用 Python 编写，可用来在不同的机器上运行同一个任务，使用 NFS 作为文件系统的通讯。更新内容：修复exit_after_n_tasks中的错误。（详情：https://github.com/luispedro/jug/archive/v1.6.4.zip）

2、PHP 开发框架 CakePHP 3.5.5 发布

CakePHP 3.5.5 发布了。CakePHP 是一个运用了诸如 ActiveRecord、Association Data Mapping、Front Controller 和 MVC 等著名设计模式的快速开发框架。主要更新内容： SQLServer no longer reports double column lengths for NVARCHAR and NCHAR columns.（详情：https://github.com/cakephp/cakephp/compare/3.5.4...3.5.5）

3、Swoole 1.9.22 发布，增加 Async::exec 功能

PHP的异步、并行、高性能网络通信引擎 Swoole 已发布 1.9.22 版本。此版本增加了 Async::exec 功能，可以异步执行一条 shell 命令。另外此版本还修复了多个已知问题，如修复MacOS 下压力测试发生死循环的问题（详情：https://github.com/swoole/swoole-src/releases/tag/v1.9.22）

4、GitLab 发布 10.1.1 补丁版本

GitLab 发布 10.1.1 补丁版本，包括社区版和企业版。该版本主要是 Bug 的修复，包括： ● CE/EE: Allow boards as top level route (!14861)
● CE/EE: Forbid the usage of Redis#keys (!14889)
● ......（详情：https://gitlab.com/gitlab-org/gitlab-ce/merge_requests/）

5、高性能 PHP 框架 Yii2 2.0.13 发布

Yii2 2.0.13 发布了。是最流行的 PHP 框架之一。主要更新内容：
● yii\web\DbSession 现依靠错误处理程序来显示错误。
● 允许移动任何大小的名称，但长度限制在 180 个字符
● 日志消息时间精确到毫秒（详情：http://www.yiiframework.com/news/149/yii-2-0-13-is-released/）

6、PyCharm 2017.3.EAP 8 发布

PyCharm 2017.3.EAP 8 发布了。PyCharm 是由 JetBrains 打造的一款 Python IDE 。主要更新内容如下：
● A critical bug that resulted in some users having to wait 5-10 seconds until their script started has been resolved（详情：https://confluence.jetbrains.com/display/PYH/PyCharm+173.3531.8+Release+Notes）

7、苹果将开放更多 iPhone X 面部数据给开发者

11月3日凌晨，苹果公司或将 iPhone X 的面部识别数据与开发者共享。据媒体报道，苹果允许在获得用户批准后，开发者可访问“某些面部数据”，包括虚拟面部表情等。对此，有人担心一旦苹果放宽权限，它将不能有效地控制开发者怎样使用这些数据，这存在潜在的隐私泄漏风险。对此，苹果还未发表任何评论。
